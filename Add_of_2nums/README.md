# 两数之和

原题：[1. 两数之和 - 力扣（LeetCode）](https://leetcode.cn/problems/two-sum/description/?envType=study-plan-v2&envId=top-100-liked)

## 做题过程：

* main.c：两个for循环
* Hash.c：哈希表算法

​		拿到这题首先第一个想到的方法就是穷举法：用两个for循环遍历查找，对每个数组的元素一个个查找相加是否等于target的值。这样做在逻辑上非常朴素（笨方法），但是问题在于这个计算量为O(n^2)，冗余繁琐，在leetcode上只战胜了28.6%的对手，这让我十分沮丧。

​		后面我看了题解后学习到了一种有意思的算法——哈希表。

​		什么是哈希表？哈希表相当于是一个函数，**将无穷量化**。其来源也十分生动有趣——将马铃薯切成丝状，此时我们可以称有无数薯条（无穷），然后我们将薯条混在一起油炸，就可以获得薯饼（可量化）。在实际应用中，哈希表常常用来**校对**。比方说，我们要练习英语单词，此时抄写一页英语词典，如何用简单的方法尽量确保我们没抄错？一个方法就是根据该页标的单词数量，与我们抄写的单词数量进行比对，假设英语词典标记该页单词量有100个，我们数我们抄写的单词数量也是100个，那此时我们就有一定的把握称我们的抄写是没问题的。这样看来，哈希表的本质就是**映射**

​		当然，哈希表并**不能保证单射**条件，这意味着不同的原因指向相同的结果，发生冲突。限于笔者水平，本人暂时还没有理解解决冲突的方式，在本题就不多赘述。

​		本题的哈希表应用思路为，设当前正在检查的输入数组内数字为x，我们要找到输入数组内是否存在target-x。首先先查找哈希表内是否存在target-x的数字，如果没有，再查找哈希表内是否存在与x相同的数字，如果没有，就将x存放至表内。接着到输入数组内下个数字，重复上述过程。



## 一些编程内的感悟

`malloc`很适合用在给子函数输入数组大小，或者用于定义结构体数组时，分配动态空间。哈希表的大小是动态变化的，无法预先确定需要多少个节点。通过 `malloc` 动态分配内存，可以在需要时创建新节点，而无需提前声明固定大小的数组。
